    twitternlp

    The map above shows all tweets made on Twitter that describe a road being flooded. (Since San Francisco rarely floods, most of these heatmap pointers are just tests.) twitternlp scans Twitter for "flooding" terms within tweets and extracts street names and other location information from geolocation metadata when available. Once verified, twitternlp uses the Google Maps API to plot a heatmap.

    How does it work?
  • Tokenization and Grammatical Structure Analysis

    Natural Language Processing

    As a tweet enters the twitternlp webapp, it is first cleaned and normalized. With internet lingo removed, it is initially parsed for certain keywords. If found, sequencing of certain grammatical attributes are compared and contrasted. Probabilities are assigned to combinations of "parts of speech" tags and sent through additional verification functions until twitternlp thinks it has found the correct street name.

    Show me the code!
  • The pipeline behind the scenes

    Data Engineering

    The entire process of scanning Twitter and performing NLP processes is carried out by Python. A Python script geocodes the discovered location into coordinates and sends them off into a database. The Javascript function within this webpage will sense an update to the database, pull all new values, and update the static map above.

Live in SF? Try It Out!

Location

Currently, only San Francisco is being scanned. This is so that I can easily add features and test the geolocation parsing of some of the functions. However, if you have a geolocation enabled device with the Twitter App and would like to see twitternlp in action, please contact me! I can quickly reconfigure the code to your area.

Using Twitter

Using your Twitter App (Currently twitternlp is only configured for mobile devices) start writing a tweet that would replicate the circumstances of a person posting about a flooded street. Some examples: "OMG Market St is flooding!!", "I just came back from the gym and saw that Tehama St is completely flooded!!!", etc.

Behind the Scenes

After submitting your tweet, twitternlp will analyze your tweet and extract the street name. A Python script finds the coordinates of the center location of your street using the Google Maps API. Next, it sends them to a NoSQL database where they are separated into latitude and longitude.

Front End

A Javascript function embedded within this webapp is constantly listening to the database. Whenever new values are added, they are instantly pulled. The static map feature of the Google Maps API then plots a heatpoint for that center coordinate. The more people tweet that street, the more red the marker will become.

Initially, this idea came about after seeing the hurricanes in the last couple of months ravage through Texas and Puerto Rico. The news kept mentioning how certain reservoirs were overflowing and unexpectedly flooding streets in a short period of time. This, already adding to the disaster that the heavy rains and strong winds were causing. I figured a website like this would enable people to avoid deadly situations and quickly have a glimpse of the area around them to save friends or loved ones. If even just one person finds this concept helpful in the future, then it was worth it.

I plan on opening this up to scan the entire United States, and eventually writing up some algorithms that can over time learn which locations to isolate based on trending semantics. A Word2Vec model for greater variety of flooding indications in tweets, is in the near future as well. If you'd like to contribute to this project, feel free to contact me.
